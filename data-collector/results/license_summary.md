# TMDB ToS 핵심 제약 정리

> 출처: [TMDB Terms of Use](https://www.themoviedb.org/documentation/api/terms-of-use) (2024년 기준)
> 참조: [TMDB API FAQ](https://developer.themoviedb.org/docs/faq)

---

## 1. 캐싱 정책

| 항목 | 제약 |
|------|------|
| 캐시 허용 기간 | **최대 30일** (일반 데이터) |
| 이미지 캐시 | 허용, 단 TMDB 귀속 표시 유지 |
| 갱신 주기 | 변경된 ID 목록 API로 증분 동기화 권장 |

> ⚠️ 본 시스템은 6개월 캐시 TTL을 사용하도록 설계되었습니다. **ToS 30일 제한과 불일치**합니다. 상용 서비스 전 법무 검토가 필요합니다.

---

## 2. AI/ML 사용 금지

- **TMDB 데이터를 AI/ML 모델 학습에 직접 사용하는 것은 금지**됩니다.
- 추천 엔진의 feature로 TMDB 메타데이터를 활용하는 경우 ToS 해석이 모호합니다.
- 사용 목적이 "영화 검색 및 메타데이터 표시"에 한정된다면 허용 범위 내입니다.

---

## 3. 상업화 조건

- **비상업적 개인 프로젝트**: 무료 사용 가능
- **상업적 사용**: TMDB에 별도 문의 및 계약 필요
- 앱/서비스에 TMDB 데이터를 표시할 경우 "This product uses the TMDB API but is not endorsed or certified by TMDB." 문구 표시 의무

---

## 4. API 요청 제한

- **공식 Rate Limit**: 초당 40 요청 (v3 기준)
- 본 시스템 설정: `API_CONCURRENCY = 4` (동시 요청 4건, 보수적 운영)
- 일일 요청 한도: 명시 없음, 과도한 사용 시 키 차단 가능

---

## 5. 데이터 귀속 표시

- 모든 TMDB 데이터 표시 화면에 TMDB 로고 또는 귀속 문구 필수
- 이미지: TMDB CDN URL 직접 사용 또는 귀속 표시와 함께 캐시 허용

---

## 6. 금지 사항 요약

| 금지 항목 | 내용 |
|-----------|------|
| 데이터 재배포 | TMDB 데이터를 별도 데이터셋으로 배포 불가 |
| AI 학습 | 모델 학습 데이터로 직접 사용 금지 |
| 귀속 미표시 | TMDB 출처 표시 없는 데이터 노출 불가 |
| 장기 캐싱 | 30일 초과 캐시 (ToS 해석에 따라 다름) |

---

## 7. 결론 및 권고사항

현재 cine-mirror 설계의 **ToS 위험 요소**:

1. **6개월 캐시 TTL** → 30일 제한 초과. 주기적 갱신 로직 보완 또는 30일 TTL로 단축 검토 필요.
2. **AI 추천 활용 의도** → TMDB 데이터를 추천 feature로 사용 시 사전 허가 필요.
3. **상업화 계획** → 유료 서비스 전환 시 TMDB 파트너십 또는 대안 데이터 소스 검토 필요.
